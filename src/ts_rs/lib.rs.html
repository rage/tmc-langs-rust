24//! </a>
25//! </div>
26//!
27//! ## why?
28//! When building a web application in rust, data structures have to be shared between backend and frontend.
29//! Using this library, you can easily generate TypeScript bindings to your rust structs & enums so that you can keep your
30//! types in one place.
31//!
32//! ts-rs might also come in handy when working with webassembly.
33//!
34//! ## how?
35//! ts-rs exposes a single trait, `TS`. Using a derive macro, you can implement this interface for your types.
36//! Then, you can use this trait to obtain the TypeScript bindings.
37//! We recommend doing this in your tests.
38//! [See the example](https://github.com/Aleph-Alpha/ts-rs/blob/main/example/src/lib.rs) and [the docs](https://docs.rs/ts-rs/latest/ts_rs/).
39//!
40//! ## get started
41//! ```toml
42//! [dependencies]
43//! ts-rs = "7.1"
44//! ```
45//!
46//! ```rust
47//! use ts_rs::TS;
48//!
49//! #[derive(TS)]
50//! #[ts(export)]
51//! struct User {
52//!     user_id: i32,
53//!     first_name: String,
54//!     last_name: String,
55//! }
56//! ```
57//! When running `cargo test`, the TypeScript bindings will be exported to the file `bindings/User.ts`.
58//!
59//! ## features
60//! - generate interface declarations from rust structs
61//! - generate union declarations from rust enums
62//! - inline types
63//! - flatten structs/interfaces
64//! - generate necessary imports when exporting to multiple files
65//! - serde compatibility
66//! - generic types
67//! - support for ESM imports
68//!
69//! ## limitations
70//! - generic fields cannot be inlined or flattened (#56)
71//! - type aliases must not alias generic types (#70)
72//!
73//! ## cargo features
74//! - `serde-compat` (default)  
75//!
76//!   Enable serde compatibility. See below for more info.  
77//! - `format`
78//!
79//!   When enabled, the generated typescript will be formatted.
80//!   Currently, this sadly adds quite a bit of dependencies.
81//! - `chrono-impl`  
82//!
83//!   Implement `TS` for types from chrono  
84//! - `bigdecimal-impl`  
85//!
86//!   Implement `TS` for types from bigdecimal  
87//! - `url-impl`  
88//!
89//!   Implement `TS` for types from url
90//! - `uuid-impl`  
91//!
92//!   Implement `TS` for types from uuid
93//! - `bson-uuid-impl`
94//!
95//!   Implement `TS` for types from bson
96//! - `bytes-impl`
97//!
98//!   Implement `TS` for types from bytes    
99//! - `indexmap-impl`  
100//!
101//!   Implement `TS` for `IndexMap` and `IndexSet` from indexmap  
102//!
103//! - `ordered-float-impl`  
104//!
105//!   Implement `TS` for `OrderedFloat` from ordered_float
106//!
107//! - `heapless-impl`  
108//!
109//!   Implement `TS` for `Vec` from heapless
110//!
111//! - `semver-impl`  
112//!   Implement `TS` for `Version` from semver
113//!
114//! - `no-serde-warnings`
115//!
116//!   When `serde-compat` is enabled, warnings are printed during build if unsupported serde
117//!   attributes are encountered. Enabling this feature silences these warnings.
118//!
119//! - `import-esm`
120//!
121//!   `import` statements in the generated file will have the `.js` extension in the end of
122//!   the path to conform to the ES Modules spec. (e.g.: `import { MyStruct } from "./my_struct.js"`)
123//!
124//! If there's a type you're dealing with which doesn't implement `TS`, use `#[ts(type = "..")]` or open a PR.
125//!
126//! ## serde compatability
127//! With the `serde-compat` feature (enabled by default), serde attributes can be parsed for enums and structs.
128//! Supported serde attributes:
129//! - `rename`
130//! - `rename-all`
131//! - `rename-all-fields`
132//! - `tag`
133//! - `content`
134//! - `untagged`
135//! - `skip`
136//! - `skip_serializing`
137//! - `skip_deserializing`
138//! - `skip_serializing_if = "Option::is_none"`
139//! - `flatten`
140//! - `default`
141//!
142//! Note: `skip_serializing` and `skip_deserializing` are ignored. If you wish to exclude a field
143//! from the generated type, but cannot use `#[serde(skip)]`, use `#[ts(skip)]` instead.
144//!
145//! When ts-rs encounters an unsupported serde attribute, a warning is emitted, unless the feature `no-serde-warnings` is enabled.
146//!

189/// A type which can be represented in TypeScript.  
190/// Most of the time, you'd want to derive this trait instead of implementing it manually.  
191/// ts-rs comes with implementations for all primitives, most collections, tuples,
192/// arrays and containers.
193///
194/// ### exporting
195/// Because Rusts procedural macros are evaluated before other compilation steps, TypeScript
196/// bindings cannot be exported during compile time.
197/// Bindings can be exported within a test, which ts-rs generates for you by adding `#[ts(export)]`
198/// to a type you wish to export to a file.
199/// If, for some reason, you need to do this during runtime, you can call [`TS::export`] yourself.
200///
201/// ### serde compatibility
202/// By default, the feature `serde-compat` is enabled.
203/// ts-rs then parses serde attributes and adjusts the generated typescript bindings accordingly.
204/// Not all serde attributes are supported yet - if you use an unsupported attribute, you'll see a
205/// warning.
206///
207/// ### container attributes
208/// attributes applicable for both structs and enums
209///
210/// - `#[ts(export)]`:  
211///   Generates a test which will export the type, by default to `bindings/<name>.ts` when running
212///   `cargo test`. The default base directory can be overridden with the `TS_RS_EXPORT_DIR` environment variable.
213///   Adding the variable to a project's [config.toml](https://doc.rust-lang.org/cargo/reference/config.html#env) can
214///   make it easier to manage.
215/// ```toml
216/// # <project-root>/.cargo/config.toml
217/// [env]
218/// TS_RS_EXPORT_DIR = { value = "<OVERRIDE_DIR>", relative = true }
219/// ```
220///
221/// - `#[ts(export_to = "..")]`:  
222///   Specifies where the type should be exported to. Defaults to `bindings/<name>.ts`.  
223///   The `export_to` attribute will also override the `TS_RS_EXPORT_DIR` environment variable.  
224///   If the provided path ends in a trailing `/`, it is interpreted as a directory.   
225///   Note that you need to add the `export` attribute as well, in order to generate a test which exports the type.
226///
227/// - `#[ts(rename = "..")]`:  
228///   Sets the typescript name of the generated type
229///
230/// - `#[ts(rename_all = "..")]`:  
231///   Rename all fields/variants of the type.
232///   Valid values are `lowercase`, `UPPERCASE`, `camelCase`, `snake_case`, `PascalCase`, `SCREAMING_SNAKE_CASE`, "kebab-case"
233///
234///
235/// ### struct field attributes
236///
237/// - `#[ts(type = "..")]`:  
238///   Overrides the type used in TypeScript.  
239///   This is useful when there's a type for which you cannot derive `TS`.  
240///
241/// - `#[ts(rename = "..")]`:  
242///   Renames this field  
243///
244/// - `#[ts(inline)]`:  
245///   Inlines the type of this field  
246///
247/// - `#[ts(skip)]`:  
248///   Skip this field  
249///
250/// - `#[ts(optional)]`:  
251///   May be applied on a struct field of type `Option<T>`.
252///   By default, such a field would turn into `t: T | null`.
253///   If `#[ts(optional)]` is present, `t?: T` is generated instead.
254///   If `#[ts(optional = nullable)]` is present, `t?: T | null` is generated.
255///
256/// - `#[ts(flatten)]`:  
257///   Flatten this field
258///   
259/// ### enum attributes
260///
261/// - `#[ts(tag = "..")]`:  
262///   Changes the representation of the enum to store its tag in a separate field.
263///   See [the serde docs](https://serde.rs/enum-representations.html).
264///
265/// - `#[ts(content = "..")]`:  
266///   Changes the representation of the enum to store its content in a separate field.
267///   See [the serde docs](https://serde.rs/enum-representations.html).
268///
269/// - `#[ts(untagged)]`:  
270///   Changes the representation of the enum to not include its tag.
271///   See [the serde docs](https://serde.rs/enum-representations.html).
272///
273/// - `#[ts(rename_all = "..")]`:  
274///   Rename all variants of this enum.  
275///   Valid values are `lowercase`, `UPPERCASE`, `camelCase`, `snake_case`, `PascalCase`, `SCREAMING_SNAKE_CASE`, "kebab-case"
276///
277/// - `#[ts(rename_all_fieds = "..")]`
278///   Renames the fields of all the struct variants of this enum.
279///   Valid values are `lowercase`, `UPPERCASE`, `camelCase`, `snake_case`, `PascalCase`, `SCREAMING_SNAKE_CASE`, "kebab-case"
280///  
281/// ### enum variant attributes
282///
283/// - `#[ts(rename = "..")]`:  
284///   Renames this variant  
285///
286/// - `#[ts(skip)]`:  
287///   Skip this variant  
288pub trait TS {
289    const EXPORT_TO: Option<&'static str> = None;
290    const DOCS: Option<&'static str> = None;
291
292    fn get_export_to() -> Option<String> {
293        Self::EXPORT_TO.map(ToString::to_string)
294    }
295
296    /// Declaration of this type, e.g. `interface User { user_id: number, ... }`.
297    /// This function will panic if the type has no declaration.
298    fn decl() -> String {
299        panic!("{} cannot be declared", Self::name());
300    }
301
302    /// Name of this type in TypeScript.
303    fn name() -> String;
304
305    /// Name of this type in TypeScript, with type arguments.
306    fn name_with_type_args(args: Vec<String>) -> String {
307        format!("{}<{}>", Self::name(), args.join(", "))
308    }
309
310    /// Formats this types definition in TypeScript, e.g `{ user_id: number }`.
311    /// This function will panic if the type cannot be inlined.
312    fn inline() -> String {
313        panic!("{} cannot be inlined", Self::name());
314    }
315
316    /// Flatten an type declaration.  
317    /// This function will panic if the type cannot be flattened.
318    fn inline_flattened() -> String {
319        panic!("{} cannot be flattened", Self::name())
320    }
321
322    fn dependency_types() -> impl TypeList
323    where
324        Self: 'static,
325    {
326    }
327
328    fn dependencies() -> Vec<Dependency>
329    where
330        Self: 'static,
331    {
332        use crate::typelist::TypeVisitor;
333
334        let mut deps: Vec<Dependency> = vec![];
335        struct Visit<'a>(&'a mut Vec<Dependency>);
336        impl<'a> TypeVisitor for Visit<'a> {
337            fn visit<T: TS + 'static + ?Sized>(&mut self) {
338                if let Some(dep) = Dependency::from_ty::<T>() {
339                    self.0.push(dep);
340                }
341            }
342        }
343        Self::dependency_types().for_each(&mut Visit(&mut deps));
344
345        deps
346    }
347
348    /// `true` if this is a transparent type, e.g tuples or a list.
349    /// This is used for resolving imports when using the `export!` macro.
350    fn transparent() -> bool;
351
352    /// Manually export this type to a file.
353    /// The output file can be specified by annotating the type with `#[ts(export_to = ".."]`.
354    /// By default, the filename will be derived from the types name.
355    ///
356    /// When a type is annotated with `#[ts(export)]`, it is exported automatically within a test.
357    /// This function is only usefull if you need to export the type outside of the context of a
358    /// test.
359    fn export() -> Result<(), ExportError>
360    where
361        Self: 'static,
362    {
363        export::export_type_with_dependencies::<Self>()
364    }
365
366    /// Manually export this type to a file with a file with the specified path. This
367    /// function will ignore the `#[ts(export_to = "..)]` attribute.
368    fn export_to(path: impl AsRef<Path>) -> Result<(), ExportError>
369    where
370        Self: 'static,
371    {
372        export::export_type_to::<Self, _>(path)
373    }
374
375    /// Manually generate bindings for this type, returning a [`String`].  
376    /// This function does not format the output, even if the `format` feature is enabled.
377    fn export_to_string() -> Result<String, ExportError>
378    where
379        Self: 'static,
380    {
381        export::export_type_to_string::<Self>()
382    }
383}

385/// A typescript type which is depended upon by other types.
386/// This information is required for generating the correct import statements.
387#[derive(Debug, Eq, PartialEq, Ord, PartialOrd)]
388pub struct Dependency {
389    /// Type ID of the rust type
390    pub type_id: TypeId,
391    /// Name of the type in TypeScript
392    pub ts_name: String,
393    /// Path to where the type would be exported. By default a filename is derived from the types
394    /// name, which can be customized with `#[ts(export_to = "..")]`.
395    pub exported_to: String,
396}
397
398impl Dependency {
399    /// Constructs a [`Dependency`] from the given type `T`.
400    /// If `T` is not exportable (meaning `T::EXPORT_TO` is `None`), this function will return
401    /// `None`
402    pub fn from_ty<T: TS + 'static + ?Sized>() -> Option<Self> {
403        let exported_to = T::get_export_to()?;
404        Some(Dependency {
405            type_id: TypeId::of::<T>(),
406            ts_name: T::name(),
407            exported_to,
408        })
409    }
410}

412// generate impls for primitive types
413macro_rules! impl_primitives {
414    ($($($ty:ty),* => $l:literal),*) => { $($(
415        impl $crate::TS for $ty {
416            fn name() -> String { $l.to_owned() }
417            fn name_with_type_args(args: Vec<String>) -> String {
418                assert!(args.is_empty(), "called name_with_type_args on primitive");
419                $l.to_owned()
420            }
421            fn inline() -> String { $l.to_owned() }
422            fn transparent() -> bool { false }
423        }
424    )*)* };
425}
426// generate impls for tuples
427macro_rules! impl_tuples {
428    ( impl $($i:ident),* ) => {
429        impl<$($i: TS),*> TS for ($($i,)*) {
430            fn name() -> String {
431                format!("[{}]", [$($i::name()),*].join(", "))
432            }
433            fn inline() -> String {
434                format!("[{}]", [$($i::inline()),*].join(", "))
435            }
436            fn dependency_types() -> impl TypeList
437            where
438                Self: 'static
439            {
440                ()$(.push::<$i>())*
441            }
442            fn transparent() -> bool { true }
443        }
444    };
445    ( $i2:ident $(, $i:ident)* ) => {
446        impl_tuples!(impl $i2 $(, $i)* );
447        impl_tuples!($($i),*);
448    };
449    () => {};
450}

452// generate impls for wrapper types
453macro_rules! impl_wrapper {
454    ($($t:tt)*) => {
455        $($t)* {
456            fn name() -> String { T::name() }
457            fn name_with_type_args(mut args: Vec<String>) -> String {
458                assert_eq!(args.len(), 1);
459                args.remove(0)
460            }
461            fn inline() -> String { T::inline() }
462            fn inline_flattened() -> String { T::inline_flattened() }
463            fn dependency_types() -> impl TypeList
464            where
465                Self: 'static
466            {
467                T::dependency_types()
468            }
469            fn transparent() -> bool { T::transparent() }
470        }
471    };
472}

562// Arrays longer than this limit will be emitted as Array<T>
563const ARRAY_TUPLE_LIMIT: usize = 64;
564impl<T: TS, const N: usize> TS for [T; N] {
565    fn name() -> String {
566        if N > ARRAY_TUPLE_LIMIT {
567            return Vec::<T>::name();
568        }
569
570        "[]".to_owned()
571    }
572
573    fn name_with_type_args(args: Vec<String>) -> String {
574        if N > ARRAY_TUPLE_LIMIT {
575            return Vec::<T>::name_with_type_args(args);
576        }
577
578        assert_eq!(
579            args.len(),
580            1,
581            "called [T; N]::name_with_type_args with {} args",
582            args.len()
583        );
584
585        format!(
586            "[{}]",
587            (0..N)
588                .map(|_| args[0].clone())
589                .collect::<Box<[_]>>()
590                .join(", ")
591        )
592    }
593
594    fn inline() -> String {
595        if N > ARRAY_TUPLE_LIMIT {
596            return Vec::<T>::inline();
597        }
598
599        format!(
600            "[{}]",
601            (0..N).map(|_| T::inline()).collect::<Box<[_]>>().join(", ")
602        )
603    }
604
605    fn dependency_types() -> impl TypeList
606    where
607        Self: 'static,
608    {
609        ().push::<T>()
610    }
611
612    fn transparent() -> bool {
613        true
614    }
615}
